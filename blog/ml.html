How models actually learn

Every ML model, from linear regression to GPT-4, learns the same way: it makes a guess, measures how wrong it was, and adjusts. Here's the trick in one interactive chart.

The setup: supervised learning in one paragraph

You have inputs x and outputs y. You want a function f(x) ≈ y. ML says: pick a flexible function family (line, polynomial, neural net), define how wrong it is (the loss function), and tune its parameters until the loss is small. Done. That's all machine learning.

The simplest case: fitting a line

Given a cloud of points, find the line y = m·x + b that sits closest to them. "Closest" typically means minimum mean squared error (MSE): average of (y_predicted - y_actual)² across all points.

Click the canvas below to drop points. The line refits instantly using the closed-form ordinary-least-squares solution. The dashed pink lines show the residuals — the errors the line is trying to minimise.

Why gradient descent beats closed-form

For a line you can solve it with algebra in one step. But once your function has millions of parameters (neural nets), algebra gives up. Instead we use gradient descent:

  1. Start with random weights.
  2. Compute the loss.
  3. Compute the gradient — which direction each weight should move to reduce the loss.
  4. Take a small step in that direction. Repeat millions of times.

It's literally walking downhill on a loss surface. The "learning rate" is your step size — too big and you overshoot, too small and it takes forever.

Every model you've heard of — ResNet, BERT, GPT-4, Claude, Gemini — is just gradient descent on a very large, very weird loss surface.

The two enemies: bias and variance

Fit a straight line to curvy data → underfitting (high bias). The model is too simple to capture the pattern.

Fit a 100-degree polynomial through 10 points → overfitting (high variance). The model memorises noise instead of the signal. Hand it new data and it falls apart.

Everything in ML — regularisation, dropout, cross-validation, early stopping — is fighting this trade-off. The goal: generalise, not memorise.

Core recipes you should know

  • Linear / logistic regression — your baseline. Always fit one first.
  • Gradient boosted trees (XGBoost, LightGBM) — still crushing tabular data in 2026.
  • CNNs — images, spatial data.
  • Transformers — text, code, protein folding, you name it.
  • Contrastive learning — how modern embeddings get trained (CLIP, SimCLR).

What this has to do with my research

In LLM trustworthiness, we fight overfitting in a weird new way: the model doesn't overfit its training data, but it overfits the shape of plausible text. Result: confident hallucinations. In the next post I explain why, and what we're doing about it at CAREAI Lab.
