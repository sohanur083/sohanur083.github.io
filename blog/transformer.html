#05 · Transformer Deep-Dive

How transformers actually work

Every modern LLM — GPT, Claude, Llama, Gemini — is a transformer. The 2017 paper "Attention Is All You Need" replaced recurrence with attention and changed the field. Here's what's actually happening inside that stack.

1. The one-slide version

A transformer is a stack of identical blocks. Each block has two sub-layers: multi-head self-attention, and a feed-forward network. Residual connections and layer norm wrap both. Input goes in as token embeddings, output comes out as contextualised embeddings. Stack 12 of these → BERT. Stack 96 → GPT-3. Stack 80 with some tweaks → Llama 3.

2. Self-attention in one equation

For every token, we produce three vectors: Query (Q), Key (K), Value (V). Each is just a learned linear projection of the token embedding. Attention weights = how much each Query matches each Key: 

Attention(Q, K, V) = softmax( Q · Kᵀ / √d_k ) · V

That's it. The √d_k is a scaling factor to keep gradients stable. Softmax turns raw scores into a probability distribution over the sequence. Multiply by V and you get a weighted sum of value vectors — the new representation for each token.

The magic: every token sees every other token in one step. RNNs had to unroll sequentially. Attention is parallel. This is why transformers train so fast on GPUs.

3. Why multi-head?

One attention head can learn only one kind of relationship. Multiple heads run in parallel, each with its own Q/K/V projections. Some heads learn syntactic patterns (previous word, subject-verb), others learn semantic (coreference, entity linking), others learn long-range dependencies. Their outputs are concatenated and mixed.

Different heads learn genuinely different skills. Analysing them is its own research area — see Anthropic's circuits work, or Elhage et al. on induction heads that seem to underlie in-context learning.

4. Positional encoding — how order gets in

Attention is order-invariant. Shuffle the tokens, output is identical. That's fatal for language. Fix: add a position signal to each token embedding.

  • Sinusoidal (original paper) — fixed sin/cos waves at different frequencies.
  • Learned (BERT, early GPT) — a separate embedding per position, learned from data.
  • RoPE (Llama, Qwen, most modern LLMs) — rotary position embeddings. Baked into Q and K themselves. Generalises better to longer sequences.
  • ALiBi (BLOOM, MPT) — linear bias on attention scores based on distance. Extrapolates to unseen lengths.

5. Feed-forward block: where knowledge lives

After attention, each token goes through a 2-layer MLP independently. Usually 4× the hidden dimension. It's huge: in GPT-3 the FFN accounts for ~two-thirds of all parameters.

Recent work (Geva et al.) argues the FFN layers act like a key-value store of facts. When you ask GPT "the capital of France is ___", the attention routes the query to the right FFN layer, which produces "Paris." Knowledge lives in the FFN. Attention just moves it around.

6. The full LLM: how it gets built

  1. Tokeniser — BPE/SentencePiece splits text into sub-word tokens. Vocab size 32k–256k.
  2. Embedding table — vocab × d_model. GPT-3: 50k × 12288 = 600M params just here.
  3. N transformer blocks — each ~12 × d_model² params. Llama-7B: 32 blocks, d_model=4096.
  4. Output head — linear projection back to vocab size. Often tied to embedding table to save params.
  5. Pre-training — next-token prediction on trillions of tokens of web text. Takes weeks on thousands of GPUs.
  6. Post-training — supervised fine-tuning (SFT) + RLHF / DPO on human preferences to make it helpful and safe.
A 7B model has 7 billion parameters, each stored in fp16 (2 bytes). That's 14 GB just to load. Training requires roughly 5× that for gradients + optimiser state. That's why fine-tuning is expensive — and why we invented LoRA / QLoRA.

7. Types of LLMs — not all transformers are equal

By architecture

  • Encoder-only (BERT, RoBERTa) — bi-directional attention. Great for classification, embeddings, search. Can't generate.
  • Decoder-only (GPT, Claude, Llama, Gemini) — causal / masked attention, each token only sees past tokens. Generates text. The dominant paradigm.
  • Encoder-decoder (T5, BART, FLAN-T5) — encoder processes input fully, decoder generates output. Strong for translation, summarisation.

By training regime

  • Base models — just next-token prediction. Raw, not conversational.
  • Instruct models — base + supervised fine-tuning on instruction-response pairs. Follows commands.
  • Chat models — instruct + RLHF/DPO. Polite, safe, conversational. ChatGPT, Claude.
  • Reasoning models (o1, Claude thinking, DeepSeek-R1) — RL-tuned to generate explicit chain-of-thought before final answer. Slow but smart.

By scale + openness

  • Frontier closed-source — GPT-4/5, Claude Opus, Gemini Ultra. 100B+ params (estimated). Proprietary APIs only.
  • Open-weight — Llama 3, Qwen, Mistral, DeepSeek. 7B–405B. Weights published, run locally.
  • Mixture-of-Experts (MoE) — DeepSeek-V3, Mixtral, GPT-4 (rumoured). Many "expert" FFN modules, only a few activate per token. Huge param count, cheap inference.
  • Small/edge — Phi-3, Gemma 2B, TinyLlama. Cheap, run on laptops, good enough for many tasks.

8. What I actually care about

As a researcher on trustworthy AI, the part that fascinates me is: all this beautiful math still confidently hallucinates. The attention patterns look reasonable, the FFN stores facts, RLHF makes it polite — and yet when you ask a medical LLM for a citation, half the time it makes one up. Understanding why is the open problem. That's where explainability and hallucination research come in.
